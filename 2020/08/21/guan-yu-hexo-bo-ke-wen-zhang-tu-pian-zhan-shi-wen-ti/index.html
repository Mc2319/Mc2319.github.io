关于hexo博客文章图片展示问题

关于图片展示
博客搭建
Publish Date:   2020-08-21
Word Count:   1k
Read Count:  

博客搭建完成后，在编写博文的过程中，发现需要插入的图片在上传到服务端后不展示，经过一番百度、研究、探索，得出如下解决办法。

更改配置文件

在博客根目录下，更改配置参数；

post_asset_folder: true
#表示开始使用本地静态资源

更改完成后执行新建博客文件命令，如下：

hexo new 文件名称

添加这行配置后，在生成.md文件的同时，会生成一个同名文件夹，后续博文中用到的相关资源都可以放到该文件夹中。

引入图片方式

markdown标准语法

在Typora中编辑markdown文件时，可以直接插入图片，语法如下：

![图片别名][图片.jpg]

这样的好处是在本地Typora编辑器中直接预览。

注：在不使用任何插件的情况下，在引用图片时，需要添加文件路径，才能在本地正常预览。但在正常写博文的情况下，不要添加文件夹路径，这是因为在**hexo g命令执行后，会把.md文件渲染成为.html文件，并存放在public/年/月/日**路径下的同名文件夹中，再把所对应的的文件夹下的静态资源，拷贝到该文件夹下，添加路径会导致最后渲染后无法正常显示。

使用标签插件

使用标签插件引入图片，语法如下：

&#123;% asset_img 图片.jpg 图片别名 %&#125;

该方式无法在本地编辑器预览文章图片。

使用hexo-asset-image

为了能够在本地编辑器里预览图片同时可以在网页中正常加载图片，引入hexo-asset-image插件，该插件解决的问题就是给本地图片添加绝对路径。

安装

npm install hexo-asset-image --save

安装完成后，在本地编辑器可以直接预览图片，在执行**hexo g**生成博客html文件后，发现渲染出来的图片标签使用的是如下路径：

<img src="/年/月/日/博客同名文件夹/图片.jpg" alt="">

使用**hexo-d**上传至服务器，图片正常显示。

一些问题

在使用hexo-asset-image过程中，发现直接使用npm库中的，可能会导致BUG出现，图片在网页中无法正常加载（可能是npm库中的包未更新导致）。经过对渲染后的html文件进行查看，主要是由于路径错误问题引起的。情况如下：

<img src="/年/月/日/博客同名文件夹/年/月/日/图片.jpg" alt="">
<!-- 路径重复-->
<img src="//.com/年/月/日/博客同名文件夹/图片.jpg" alt="">
<!-- 路径错误-->
解决方案如下：

打开hexo-asset-image安装目录下index.js，替换为如下代码即可。

    if(version.length > 0 && Number(version[0]) == 3)

    else

    // In hexo 3.1.1, the permalink of "about" page is like ".../about/index.html".
    var endPos = link.lastIndexOf('/') + 1;
    link = link.substring(beginPos, endPos);

    var toprocess = ['excerpt', 'more', 'content'];
    for(var i = 0; i < toprocess.length; i++)&#123;
      var key = toprocess[i];

      var $ = cheerio.load(data[key], &#123;
        ignoreWhitespace: false,
        xmlMode: false,
        lowerCaseTags: false,
        decodeEntities: false
      &#125;);

      $('img').each(function()&#123;
        if ($(this).attr('src'))&#123;
            // For windows style path, we replace '\' to '/'.
            var src = $(this).attr('src').replace('\\', '/');
            if(!/http[s]*.*|\/\/.*/.test(src) &&
               !/^\s*\//.test(src)) &#123;
              // For "about" page, the first part of "src" can't be removed.

                return elem != '';
              &#125;);

                return elem != '' && elem != '.';
              &#125;);
              if(srcArray.length > 1)
                srcArray.shift();
              src = srcArray.join('/');
              $(this).attr('src', config.root + link + src);
